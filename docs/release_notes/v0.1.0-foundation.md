# Release Notes ÔÇö v0.1.0-foundation

**Tag**: `v0.1.0-foundation`
**Date**: 2026-07-05
**Positioning**: **Foundation release.** Not production-grade agent OS.

Personal R&D Agent OS ÔÇö first portfolio release. The trust mechanisms (privacy routing, citation validation, tool permission, trace, eval) are real and exercised on the real clawd-research corpus. User-facing flows that depend on approval / escalation / multi-turn are stubs and tracked for v0.2.

For what was tested and what wasn't, see [docs/quality_baseline_v0.1.0.md](../quality_baseline_v0.1.0.md). For what's in scope vs out, see [docs/limitations.md](../limitations.md).

### Reproducing the real corpus benchmark

Real corpus numbers are NOT reproducible from a clean checkout ÔÇö they require the local model stack and the indexed corpus:

```bash
uv run rdos doctor models  # 5/5 PASS expected
uv run rdos index-corpus --scope rag   --embedding-provider local-bge-m3 clawd-research
uv run rdos index-corpus --scope agent --embedding-provider local-bge-m3 clawd-research
uv run rdos index-corpus --scope eval  --embedding-provider local-bge-m3 clawd-research
uv run rdos benchmark retrieval --embedding-provider local-bge-m3
```

Without these steps, `rdos benchmark all` returns 0.0 on all metrics (audit-verified).

## Known limitations (highlights)

Full list in [docs/limitations.md](../limitations.md).

- **No HITL approval UI.** `requires_approval` tools return `approval_required` but cannot complete.
- **No cloud escalation.** `private_summary` cannot safely escalate; falls back to local.
- **No multi-turn conversation.** Each `ask` is independent.
- **No-answer framework exists but is disabled by default.**
- **InMemorySaver only.**
- **Only 3 of 25 clawd-research scopes are benchmarked.**
- **v0.1.0 is not production-grade.**
