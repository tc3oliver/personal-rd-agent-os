# Release Notes — v0.2.0-trust-runtime

**Tag**: `v0.2.0-trust-runtime`
**Branch**: `feat/v0.2-trust-runtime`
**Positioning**: Trust Runtime release. Closes the four trust gaps from v0.1.

## Phase 3 (Batch 19–22)

- ✅ **Batch 19** — HITL Approval Runtime. LangGraph interrupt/resume + SQLite checkpointer + `ApprovalQueue` with idempotency_key + `rdos approval list|show|approve|deny`. Decisions are immutable; replay protected.
- ✅ **Batch 20** — No-answer Calibration. 30-sample eval set (fabricated papers / concept drift / misspellings / out-of-scope) + per-collection calibrator + `no_answer_reason` in trace + separate `NO_ANSWER_GATE` (No-answer Accuracy ≥ 0.90, False No-answer Rate ≤ 0.05).
- ✅ **Batch 21** — Redaction + Cloud Escalation guardrails. 8 recognizers (EMAIL / PHONE_TW / ID_TW / URL / IP / CREDIT_CARD with Luhn / ADDRESS_TW / COMPANY_HINT) + 3 replacement strategies + `PromptPrivacyValidator` last-line-of-defense + `REDACTION_GATE` (Recall ≥ 0.95, Precision ≥ 0.95). **Note (Batch 23 audit): the actual cloud HTTP adapter is intentionally NOT shipped in v0.2/v1.0 — `cloud_send()` shim + `validate_prompt()` are tested and ready, but no sender exists. Local-first is the design constraint.**
- ✅ **Batch 22** — Multi-turn Research Thread. ThreadState + TurnRecord + follow-up rewriter (pronoun / deixis resolution) + cited context carry-forward + memory compression + `rdos thread new|ask|list|show|close`. **Note (Batch 23 audit): `rdos thread` CLI was unreachable in v0.2.0 due to a missing `add_typer` registration; fixed in v0.2.1.**
