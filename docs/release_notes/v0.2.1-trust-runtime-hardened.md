## Batch 23 — Trust Runtime Hardening

- ✅ **P0-1**: Registered `rdos thread` in main CLI. `rdos --help` now lists 14 commands; `rdos thread new|ask|list|show|close` works end-to-end.
- ✅ **P0-2**: Untracked runtime files that slipped into git history before `.gitignore` rules covered them: `data/approvals.db`, `data/samples/trace_sample.jsonl`, `data/generated/reports/synthesis_*.md`.
- ✅ **P1-1**: Wired `context_for_new_turn` into thread CLI's retrieval query. Multi-turn follow-ups now compose `original + prior topic tokens + prior cited chunk_ids` so the keyword channel sees earlier-turn context. Trace records `original_followup_query` / `rewritten_query` / `composed_retrieval_query` / `context_for_new_turn_used`.
- ✅ **P1-2**: Made `prompt_privacy_validator.validate_prompt` LIVE. New `src/rdos/llm/cloud_send.py` shim is the mandatory pre-call hook for any future cloud adapter. `cloud_send_or_raise(messages)` blocks PII with structured `PrivacyBlockError`. No cloud adapter shipped yet — but when one lands it MUST go through this shim.
- ✅ **P1-3**: Refreshed stale docs — `limitations.md` reclassified trace redaction as Implemented; `quality_baseline_v0.1.0.md` flagged `structured_output_json_validity` as formerly hardcoded (now real); `release_notes/v0.2.0-trust-runtime.md` acknowledges cloud adapter is infra-only and `rdos thread` was unreachable in v0.2.0.
- ✅ **P1-4 + P1-5**: Surfaced all Batch 13 retrieval knobs into `configs/rag.yaml` (`vector_top_k`, `keyword_top_k`, `rerank_top_k`, `min_score_threshold`, `no_answer_threshold: 0.0` explicit, `enable_query_rewrite: true`). Defaults match `RetrievalConfig` so config-driven behavior is visible without reading source.
- ✅ **P1-6**: Replaced bare `except Exception` in `cli/approval.py:154` with structured failure handling. Errors are surfaced to CLI AND recorded to `data/approvals.db` via `deny_reason` field — no silent swallow.
